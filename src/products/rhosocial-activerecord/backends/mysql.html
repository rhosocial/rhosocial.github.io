MySQL

成熟的企业级关系数据库。连接池、SSL 加密、字符集与分区完整支持。

Features

MySQL 特性。

01 / Enterprise

企业级

连接池、SSL/TLS 加密、多字符集与排序规则。支持 READ_UNCOMMITTED / READ_COMMITTED / REPEATABLE_READ / SERIALIZABLE 四种隔离级别。

02 / Upsert

原生 Upsert

ON DUPLICATE KEY UPDATE / REPLACE INTO / INSERT IGNORE 三种冲突处理策略，无需额外扩展。

03 / Partition

分区表

RANGE / LIST / HASH / KEY 分区。通过 MySQLPartitionSupport 协议格式化分区 DDL。

04 / SHOW

SHOW 命令

17 种 SHOW 表达式类，覆盖 SHOW CREATE TABLE / SHOW COLUMNS / SHOW INDEX / SHOW STATUS 等。

Connection

连接配置。

MySQLConnectionConfig 继承 ConnectionConfig，混入 SSL / Charset / Timezone 三组配置。支持环境变量。

connection_config.py
from rhosocial.activerecord.backend.impl.mysql import MySQLBackend
from rhosocial.activerecord.backend.impl.mysql.config import MySQLConnectionConfig

# 标准连接（自动使用连接池）
config = MySQLConnectionConfig(
    host='127.0.0.1',
    port=3306,
    user='app_user',
    password='secret',
    database='app_db',
)
backend = MySQLBackend(connection_config=config)
backend.connect()

# 快捷方式：直接传连接参数
backend = MySQLBackend(
    host='127.0.0.1',
    port=3306,
    user='app_user',
    password='secret',
    database='app_db',
)

# 从环境变量创建配置（前缀 MYSQL_）
config = MySQLConnectionConfig.from_env(prefix="MYSQL_")

Dialect

MySQLDialect 版本感知。

MySQLDialect 在初始化时接受版本元组，动态启用/禁用协议方法。8.0+ 是关键分界线。

dialect_version.py
from rhosocial.activerecord.backend.impl.mysql.dialect import MySQLDialect

# 默认版本 8.0.0（启用窗口函数、CTE 等）
dialect = MySQLDialect()

# 指定 5.7 版本 → 功能降级
old_dialect = MySQLDialect(version=(5, 7, 0))
old_dialect.supports_window_functions() # False
old_dialect.supports_cte() # False
old_dialect.supports_upsert() # True (ON DUPLICATE KEY UPDATE)

# 版本对应功能表
# 5.7 — ROLLUP / JSON 函数 / 生成列
# 8.0 — Window Functions / CTE / LATERAL / VALUES list
# 8.0.14 — SET TABLE option (table_option in ALTER)
# 不支持 — RETURNING / MERGE / ROLLUP(5.7 之前)

INSERT Syntax

INSERT 语法扩展。

insert_syntax.py
from rhosocial.activerecord.backend.expression import InsertExpression, ValuesSource, Literal

# ON DUPLICATE KEY UPDATE（MySQL 专有 upsert）
insert = InsertExpression(
    dialect=dialect, into='users',
    columns=['id', 'name', 'score'],
    source=ValuesSource(dialect, [[Literal(dialect, 1), Literal(dialect, 'Alice'), Literal(dialect, 95)]]),
    dialect_options={
        'on_duplicate_key_update': {'name': Literal(dialect, 'Alice'), 'score': Literal(dialect, 95)}
    }
)
# → INSERT INTO users (id, name, score) VALUES (1, 'Alice', 95)
#   ON DUPLICATE KEY UPDATE name='Alice', score=95

# REPLACE INTO（删除旧行，插入新行）
replace = InsertExpression(
    dialect=dialect, into='users',
    columns=['id', 'name'],
    source=ValuesSource(dialect, [[Literal(dialect, 1), Literal(dialect, 'Alice')]]),
    dialect_options={'replace': True}
)

# INSERT IGNORE（静默跳过冲突行）
ignore = InsertExpression(
    dialect=dialect, into='users',
    columns=['id', 'name'],
    source=ValuesSource(dialect, [[Literal(dialect, 1), Literal(dialect, 'Alice')]]),
    dialect_options={'ignore': True}
)

Partition

分区表示例。

partition_example.py
from rhosocial.activerecord.backend.impl.mysql.dialect import MySQLDialect

dialect = MySQLDialect()

# RANGE 分区
sql, params = dialect.format_partition_by_range(
    table_name='orders',
    column='created_at',
    partitions=[
        ('p2024q1', '2024-04-01'),
        ('p2024q2', '2024-07-01'),
        ('p2024q3', '2024-10-01'),
    ]
)

# LIST 分区
sql, params = dialect.format_partition_by_list(
    table_name='users',
    column='region',
    partitions=[
        ('p_asia', ['CN', 'JP', 'KR']),
        ('p_europe', ['DE', 'FR', 'GB']),
    ]
)

# HASH 分区
sql, params = dialect.format_partition_by_hash(
    table_name='logs',
    column='id',
    num_partitions=8,
)

# 动态管理分区
sql = dialect.format_add_partition(table_name='orders', name='p2024q4', value='2025-01-01')
sql = dialect.format_drop_partition(table_name='orders', name='p2024q1')

Function Example

函数使用示例。

function_example.py
from rhosocial.activerecord.backend.impl.mysql.dialect import MySQLDialect

dialect = MySQLDialect()

# JSON 路径提取
sql, params = dialect.json_extract(column='metadata', path='$.name')
# → JSON_EXTRACT(`metadata`, '$.name')

# JSON 包含检测
sql, params = dialect.json_contains(column='tags', value='"python"')
# → JSON_CONTAINS(`tags`, '"python"')

# 全文搜索
sql, params = dialect.match_against(
    columns=['title', 'body'],
    query='python async',
    mode='NATURAL_LANGUAGE_MODE'
)
# → MATCH (`title`, `body`) AGAINST ('python async' IN NATURAL LANGUAGE MODE)

# 空间距离
sql, params = dialect.st_distance(geom1='location', geom2='ST_GeomFromText(?)')
# → ST_Distance(`location`, ST_GeomFromText(?))

# SET 查询
sql, params = dialect.find_in_set(value='admin', set_column='roles')
# → FIND_IN_SET('admin', `roles`)
Transaction

事务控制。

MySQLTransactionManager 支持全部四种 SQL 标准隔离级别。默认 REPEATABLE_READ。

transaction_control.py
from rhosocial.activerecord.backend.transaction import IsolationLevel

# 标准事务（默认 REPEATABLE READ）
with backend.transaction():
    backend.execute("INSERT INTO users ...", ())

# 指定隔离级别
with backend.transaction(isolation_level=IsolationLevel.READ_COMMITTED):
    ...

# MySQL 支持的四种隔离级别
# READ_UNCOMMITTED — 可能读到未提交数据（脏读）
# READ_COMMITTED — 只读已提交数据（PostgreSQL 默认）
# REPEATABLE_READ — 可重复读（MySQL 默认，防止幻读）
# SERIALIZABLE — 最高隔离级别，完全串行化

# Savepoint 支持
with backend.transaction():
    backend.execute("INSERT ...", ())
    with backend.transaction(): # SAVEPOINT
        backend.execute("UPDATE ...", ())
        # ROLLBACK TO SAVEPOINT → 自动回滚

# 显式锁（SELECT ... FOR UPDATE / LOCK IN SHARE MODE）
# 通过查询表达式的 dialect_options 设置
Async

异步支持。

AsyncMySQLBackend / AsyncMySQLTransactionManager 懒加载，需安装 aiomysql。

async_mysql.py
from rhosocial.activerecord.backend.impl.mysql import AsyncMySQLBackend
# 需安装: pip install rhosocial-activerecord-mysql[async]

backend = AsyncMySQLBackend(
    host='127.0.0.1',
    port=3306,
    user='app_user',
    password='secret',
    database='app_db',
)
await backend.connect()

async with backend.transaction():
    result = await backend.execute("SELECT ...", ())

await backend.disconnect()

# 懒加载机制：导入同步组件不需要 aiomysql
from rhosocial.activerecord.backend.impl.mysql import MySQLBackend # ✓ 无需 aiomysql
from rhosocial.activerecord.backend.impl.mysql import AsyncMySQLBackend # ✗ 需要 aiomysql
