Transaction

后端层面事务控制，保存点与隔离级别。

Manager

TransactionManager。

TransactionManagerBase(ABC) — 共享非 I/O 逻辑。

transaction_manager.py
from rhosocial.activerecord.backend.transaction import (
    TransactionManager, AsyncTransactionManager
)

# TransactionManagerBase(ABC) — 非I/O逻辑基类
#   ├─ TransactionManager — 同步版
#   └─ AsyncTransactionManager — 异步版

# 核心方法（sync / async 对等）
# begin()               — 开启事务（level=1 真实 BEGIN，>1 SAVEPOINT）
# commit()              — 提交（level=1 真实 COMMIT，>1 RELEASE SAVEPOINT）
# rollback()            — 回滚（level=1 真实 ROLLBACK，>1 ROLLBACK TO SAVEPOINT）
# savepoint(name=None)  — 创建保存点
# release(name)         — 释放保存点
# rollback_to(name)     — 回滚到指定保存点
# transaction(...)      — 上下文管理器（自动提交/回滚）
Isolation

隔离级别。

01

READ_UNCOMMITTED

读未提交，最低隔离级别。

02

READ_COMMITTED

读已提交，大多数数据库默认。

03

REPEATABLE_READ

可重复读，MySQL 默认。

04

SERIALIZABLE

可串行化，最高隔离级别，SQLite 唯一支持。

State

事务状态。

transaction_state.py
from rhosocial.activerecord.backend.transaction import TransactionState

# TransactionState 枚举
# INACTIVE    — 无活跃事务
# ACTIVE      — 事务进行中
# COMMITTED   — 已提交
# ROLLED_BACK — 已回滚

# 状态转换：
# INACTIVE → begin() → ACTIVE → commit() → COMMITTED → INACTIVE
# ACTIVE → rollback() → ROLLED_BACK → INACTIVE

manager = backend.transaction_manager
print(manager.state)  # TransactionState.INACTIVE

Savepoint

保存点。

嵌套事务通过保存点实现：level=1 真实 BEGIN，level>1 创建 SAVEPOINT。

savepoint.py
with backend.transaction():
    # 外层事务 — 真实 BEGIN
    User(username='Alice').save()

    with backend.transaction():
        # 内层事务 — SAVEPOINT SP_1
        User(username='Bob').save()

    # 显式保存点 API
    sp = backend.transaction_manager.savepoint('my_point')
    # ... 操作 ...
    backend.transaction_manager.rollback_to(sp)
    # 回滚到 my_point，后续操作撤销
Mode

事务模式。

01 / READ_WRITE

READ_WRITE

默认模式，允许读写操作。

02 / READ_ONLY

READ_ONLY

只读模式，禁止写操作。PostgreSQL / MySQL 5.6.5+ 支持。

03 / DEFERRABLE

DEFERRABLE

PostgreSQL 特有，可延迟约束检查至事务结束。

Context Manager

上下文管理器。

transaction_context.py
# 便捷用法 — 自动提交/回滚
with backend.transaction():
    User(username='Alice').save()
    User(username='Bob').save()
    # 成功 → COMMIT，异常 → ROLLBACK

# 异步版
async with backend.transaction():
    await User(username='Alice').asave()

# 指定隔离级别和模式 — 通过 transaction_manager
with backend.transaction_manager.transaction(
    isolation_level=IsolationLevel.SERIALIZABLE,
    mode=TransactionMode.READ_ONLY
):
    users = User.query().all()

# 也可通过 ActiveRecord 模型类
with User.transaction():
    User(username='Alice').save()
Backend Diff

各后端差异。

01 / SQLite

SQLite

仅 SERIALIZABLE / READ_UNCOMMITTED；DEFERRED / IMMEDIATE / EXCLUSIVE 开始类型。

02 / MySQL

MySQL

REPEATABLE_READ 默认隔离级别。

03 / PostgreSQL

PostgreSQL

所有标准隔离级别 + DEFERRABLE + READ ONLY / READ WRITE。

04 / Oracle

Oracle

READ_COMMITTED / SERIALIZABLE。

05 / SQL Server

SQL Server

支持 SNAPSHOT 隔离级别。

Errors

错误类型。

transaction_errors.py
from rhosocial.activerecord.backend.errors import (
    TransactionError,
    IsolationLevelError,
    UnsupportedTransactionModeError,
    UnsupportedIsolationLevelError
)

# TransactionError — 事务基础错误
# IsolationLevelError — 活跃事务中不可更改隔离级别
# UnsupportedTransactionModeError — 后端不支持该事务模式
#   .feature / .backend 属性
# UnsupportedIsolationLevelError — 后端不支持该隔离级别
#   .isolation_level / .backend 属性

try:
    backend.transaction_manager.isolation_level = IsolationLevel.SERIALIZABLE
except IsolationLevelError:
    print("事务进行中，不可更改隔离级别")
