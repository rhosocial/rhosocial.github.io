Backend Client

后端客户端架构，Mixin 组合模式。

MRO

Mixin 组合。

StorageBackendBase → Logging → TypeAdaption → SQLBuilding → Returning → ResultProcessing → SQLOperations → Execution → BatchExecution → ExecutionHooks → Connection → TransactionManagement

storage_backend_mro.py
from rhosocial.activerecord.backend.base import StorageBackend, AsyncStorageBackend

# StorageBackend 通过 12 层 Mixin 组合实现全部功能
# StorageBackendBase  — 配置、连接状态、类型注册
# LoggingMixin        — BackendLoggingMixin（非模型层 LoggingMixin）
# TypeAdaptionMixin   — prepare_parameters, _adapt_row_types
# SQLBuildingMixin    — SQL 构建辅助
# ReturningClauseMixin — RETURNING 子句处理
# ResultProcessingMixin — 结果构建与日志
# SQLOperationsMixin   — insert / update / delete / fetch_one / fetch_all
# ExecutionMixin       — execute / execute_many
# BatchExecutionMixin  — execute_batch_dml / execute_batch_dql
# ExecutionHooksMixin  — 游标、执行、自动提交、错误处理
# ConnectionMixin      — connection 属性、上下文管理器
# TransactionManagementMixin — begin/commit/rollback

# AsyncStorageBackend 同构，异步版 Mixin 替换 I/O 层

from rhosocial.activerecord.backend.type_adapter import (

    UUIDAdapter, EnumAdapter,
    BooleanAdapter, DecimalAdapter,
    ArrayAdapter
)

# SQLTypeAdapter @runtime_checkable 协议
# to_database(value, target_type, options) — Python → 数据库

01 / DateTime

DateTimeAdapter

datetime→str(ISO) / datetime→int(timestamp) / date→str / time→str

02 / JSON

JSONAdapter

dict→str / list→str；from_database 兼容已解析 JSONB

03 / UUID

UUIDAdapter

UUID→str；from_database 接受 str 或 bytes

04 / Enum

EnumAdapter

str→name / int→value；无默认注册（泛型）

05 / Boolean

BooleanAdapter

bool→int(1/0) / bool→str('true'/'false')

06 / Decimal

DecimalAdapter

Decimal→str / Decimal→float；支持 precision / rounding 选项

07 / Array

ArrayAdapter

list→str(json)；无默认注册（避免 JSONAdapter 冲突）

for batch_result in backend.execute_batch_dml(
    expressions,
    batch_size=100,
    commit_mode=BatchCommitMode.WHOLE,

Concurrency

并发提示。

concurrency_hint.py
from rhosocial.activerecord.backend.protocols import (
    ConcurrencyAware, ConcurrencyHint
)

# ConcurrencyAware 协议
# get_concurrency_hint() -> ConcurrencyHint | None

# ConcurrencyHint(frozen=True) 数据类
# max_concurrency: int | None  — None=无限制, 1=串行
# reason: str = ""

# SQLite 示例：文件级写锁，并发写入串行化
if isinstance(backend, ConcurrencyAware):
    hint = backend.get_concurrency_hint()
    if hint and hint.max_concurrency == 1:
        print(f"串行执行: {hint.reason}")

from rhosocial.activerecord.backend.impl.sqlite import (
    SQLiteBackend, AsyncSQLiteBackend
)

# 方式一：传入 database 字符串
backend = SQLiteBackend(database=":memory:")

backend = SQLiteBackend(
    connection_config=SQLiteConnectionConfig(
        database="app.db",
        timeout=10.0,
        uri=True
    )
)

# 异步后端（需安装 aiosqlite，懒加载）
