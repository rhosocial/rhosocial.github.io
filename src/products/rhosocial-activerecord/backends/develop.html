Overview

后端架构。

每个后端由五个核心组件构成：Backend（连接与执行）、Dialect（SQL 生成）、Config（连接配置）、TypeAdapter（类型转换）、TransactionManager（事务控制）。

01 / Backend

StorageBackend

12 Mixin 组合。6 个抽象方法。管理连接、执行 SQL、处理结果、适配类型。

02 / Dialect

SQLDialectBase

非 ABC，全部默认实现。覆盖 format_*() 即可自定义语法。33 个 Protocol 定义能力。

03 / Config

ConnectionConfig

BaseConfig + BasicConnectionMixin。7 组可选 Mixin 混入配置。

04 / Adapter

BaseSQLTypeAdapter

2 个抽象方法：_do_to_database / _do_from_database。7 个标准适配器可继承。

05 / Transaction

TransactionManager

模板方法模式。6 个 _do_*() 钩子。默认通过 backend.execute() 执行 SQL。

Config

实现 Config。

ConnectionConfig 继承 BaseConfig + BasicConnectionMixin。7 组 Mixin 可按需混入：Pool / SSL / Charset / Timezone / Version / Logging。

config_impl.py
from rhosocial.activerecord.backend.config import (
    ConnectionConfig, ConnectionPoolMixin, SSLMixin,
    CharsetMixin, TimezoneMixin, VersionMixin, LoggingMixin,
)

# 基本配置（仅 host/port/database/username/password）
class MyConfig(ConnectionConfig):
    driver_type: str = 'my_driver'

# 带连接池 + SSL + 时区
class MyFullConfig(ConnectionConfig, ConnectionPoolMixin, SSLMixin, TimezoneMixin):
    driver_type: str = 'my_driver'

# 可用 Mixin 及默认值：
# ConnectionPoolMixin: pool_size=5, pool_timeout=30, pool_pre_ping=False
# SSLMixin:            ssl_verify_cert=False, ssl_verify_identity=False
# CharsetMixin:        charset="utf8mb4"
# TimezoneMixin:       use_timezone=True
# VersionMixin:        version=None
# LoggingMixin:        raise_on_warnings=False, log_queries=False

# 也可以使用 Pydantic BaseModel（如 SQLServer）
# 或自定义 dataclass 字段
TypeAdapter

实现 TypeAdapter。

BaseSQLTypeAdapter 定义两个抽象方法：_do_to_database() 和 _do_from_database()。框架提供 7 个标准适配器。

type_adapter_impl.py
from rhosocial.activerecord.backend.type_adapter import BaseSQLTypeAdapter
from typing import Any, Type, Dict, Optional, Set

class MySpecialTypeAdapter(BaseSQLTypeAdapter):

    def __init__(self):
        super().__init__()
        # 注册 Python 类型 ↔ 数据库类型映射
        self._register_type(MyPyType, MyDbType)

    def _do_to_database(self, value: Any, target_type: Type,
                         options: Optional[Dict[str, Any]]) -> Any:
        # Python 值 → 数据库值
        return str(value)

    def _do_from_database(self, value: Any, target_type: Type,
                           options: Optional[Dict[str, Any]]) -> Any:
        # 数据库值 → Python 值
        return MyPyType(value)

# 框架提供的 7 个标准适配器（可继承扩展）
# DateTimeAdapter  — datetime/date/time ↔ str/int/float
# JSONAdapter      — dict/list ↔ str (JSON)
# UUIDAdapter      — UUID ↔ str
# EnumAdapter      — Enum ↔ str/int
# BooleanAdapter   — bool ↔ int/str
# DecimalAdapter   — Decimal ↔ str/float
# ArrayAdapter     — list ↔ str (JSON)

# 注册适配器到后端
# backend._register_adapter(MySpecialTypeAdapter())
Transaction

实现 TransactionManager。

TransactionManager 使用模板方法模式，6 个 _do_*() 钩子方法可按需覆盖。默认实现通过 backend.execute() 发送 SQL。

transaction_impl.py
from rhosocial.activerecord.backend.transaction import TransactionManager

class MyTransactionManager(TransactionManager):
    # 6 个可覆盖的钩子方法（默认通过 backend.execute 执行 SQL）

    def _do_begin(self) -> None:
        # 默认: self._backend.execute(*self._build_begin_sql())
        # 覆盖：添加 SET TRANSACTION ISOLATION LEVEL 等
        super()._do_begin()

    def _do_commit(self) -> None:
        super()._do_commit()

    def _do_rollback(self) -> None:
        super()._do_rollback()

    def _do_create_savepoint(self, name: str) -> None:
        super()._do_create_savepoint(name)

    def _do_release_savepoint(self, name: str) -> None:
        super()._do_release_savepoint(name)

    def _do_rollback_savepoint(self, name: str) -> None:
        super()._do_rollback_savepoint(name)

# 公共 API（模板方法，自动调用 _do_*）
# begin() → _do_begin()
# commit() → _do_commit()
# rollback(savepoint) → _do_rollback() / _do_rollback_savepoint()
# 嵌套事务自动使用 SAVEPOINT

Registration

注册与发现。

采用命名空间包架构（rhosocial.activerecord.backend.impl.<name>），无集中注册表。用户通过直接导入或 BackendGroup 配置使用后端。

registration.py
# 方式 1：直接导入
from rhosocial.activerecord.backend.impl.mydb import MyBackend
backend = MyBackend(connection_config=config)

# 方式 2：BackendGroup 配置
from rhosocial.activerecord.backend import BackendGroup
group = BackendGroup(
    name="main",
    models=[User, Post],
    config=MyConfig(...),
    backend_class=MyBackend,
)

# 命名空间包要求
# 1. 包路径: rhosocial.activerecord.backend.impl.mydb
# 2. pyproject.toml: packages.find(where=["src"])
# 3. __init__.py: 使用 __getattr__ 懒加载异步组件

# __init__.py 懒加载模式
def __getattr__(name):
    if name == 'AsyncMyBackend':
        from .backend.async_backend import AsyncMyBackend
        return AsyncMyBackend
    raise AttributeError(f"module {__name__!r} has no attribute {name!r}")

    dialect = MyDialect()
    sql, params = dialect.format_limit_offset(limit=10, offset=20)
    assert sql == "OFFSET 20 ROWS FETCH NEXT 10 ROWS ONLY"

def test_my_dialect_capabilities():
    dialect = MyDialect()
    assert dialect.supports_returning_clause() == True
