03 / Bind

中间数据绑定

ProcedureContext.bind() 绑定步骤间传递的数据。

procedure_params.py
from rhosocial.activerecord.backend.named_query import Procedure, ProcedureContext

class CalculateOrderTotal(Procedure):
    order_id: int       # 输入参数
    discount: float = 0.0  # 输入参数（有默认值）

    def run(self, ctx: ProcedureContext):
        # 执行命名查询，绑定结果
        ctx.execute(
            'myapp.queries.order_items',
            params={'order_id': self.order_id},
            bind='items'
        )
        # 获取标量值
        total = ctx.scalar('items', 'price')
Procedure

Procedure 基类。

procedure_runner.py
from rhosocial.activerecord.backend.named_query import (
    Procedure, ProcedureRunner, TransactionMode
)

# Procedure — 定义业务流程
#   run(ctx)           — 实现方法（必须覆写）
#   get_parameters()   — 自省类属性注解
#   static_diagram()   — 生成 Mermaid 流程图

# ProcedureRunner — 执行入口
runner = ProcedureRunner('myapp.procedures.CalculateOrderTotal')
runner.load()

# 执行
result = runner.run(
    backend=backend,
    user_params={'order_id': 42, 'discount': 0.1},
    transaction_mode=TransactionMode.AUTO
)

# ProcedureResult
#   outputs / logs / aborted / abort_reason
#   success (property) / diagram() (method)
Backend Diff

后端差异。

各数据库存储过程调用语法差异通过方言系统处理。

01 / PostgreSQL

CALL function()

PostgreSQL 使用 CALL 调用存储过程。

02 / MySQL

CALL procedure()

MySQL 使用 CALL 调用存储过程。

03 / Oracle

EXECUTE / BEGIN...END

Oracle 使用 EXECUTE 或 PL/SQL 匿名块。

04 / SQL Server

EXEC procedure

SQL Server 使用 EXEC 调用存储过程。

05 / SQLite

不支持

SQLite 不支持服务器端存储过程。
